Build a complete full-stack web application called "NAWIS AI" — 
an AI-powered school receptionist for New Al Wurood International 
School (NAWIS) in Jeddah, Saudi Arabia. This is a kiosk-style 
single-page app that will run fullscreen on a touchscreen laptop.

════════════════════════════════════
TECH STACK
════════════════════════════════════
Backend: Python with FastAPI
RAG (document search): LlamaIndex with ChromaDB vector store
AI model: Groq API (model: llama-3.3-70b-versatile)
WhatsApp alerts: CallMeBot API (simple HTTP GET request, no SDK)
Frontend: Single HTML file with vanilla JS and CSS (no React, 
no frameworks) — served by FastAPI as a static file
Serial comms: PySerial (for ESP32 LED controller via USB)

════════════════════════════════════
CONFIGURATION (hardcoded in a config.py file)
════════════════════════════════════
Create a config.py with these variables:
  GROQ_API_KEY = "PASTE_YOUR_KEY_HERE"
  CALLMEBOT_PHONE = "+966XXXXXXXXX"
  CALLMEBOT_APIKEY = "XXXXXX"
  SCHOOL_DOCS_FOLDER = "school_data"
  SERIAL_PORT = "COM3"  # Windows default, fallback gracefully if not found

════════════════════════════════════
BACKEND — app.py
════════════════════════════════════
Create a FastAPI app that does ALL of the following:

1. ON STARTUP:
   - Load all files from the "school_data/" folder (PDFs, .txt, 
     .csv, .docx files)
   - Build a LlamaIndex VectorStoreIndex from those documents 
     using ChromaDB as persistent storage
   - If ChromaDB already has the index built, load it (don't 
     rebuild every time — use persist_dir="chroma_db")
   - Connect to ESP32 via PySerial on SERIAL_PORT — if port not 
     found, log a warning and continue without crashing

2. POST /chat endpoint:
   - Accepts JSON: { "message": "user question", "language": "en" }
   - Sends ESP32 the character "T" (thinking state)
   - Queries the LlamaIndex index for relevant school document 
     chunks (top 5 most relevant)
   - Sends to Groq with this system prompt injected:
     
     """
     You are NAWIS AI, the official AI receptionist of New Al 
     Wurood International School (NAWIS) in Jeddah, Saudi Arabia. 
     You speak on behalf of the school warmly and professionally.
     
     YOUR JOB: Answer questions from parents and students about 
     the school — admissions, academics, facilities, events, 
     staff, rules, transport, results, and anything a school 
     receptionist would know.
     
     RULES:
     1. Keep answers concise — 2 to 4 sentences. Never ramble.
     2. Be warm, friendly, and professional.
     3. Use the SCHOOL CONTEXT provided below as your primary 
        source. If the context has the answer, use it.
     4. NEVER reveal specific student records, private teacher 
        contact details, internal financial details, or anything 
        that could embarrass the school.
     5. If a question is about sensitive private information, 
        or if you genuinely cannot find the answer in the 
        context, end your entire response with the word 
        ESCALATE on its own line. Do not announce you are 
        escalating — just add the word at the end.
     6. If the user's message is in Arabic, respond in Arabic.
     7. Always refer to the school as NAWIS or Al Wurood.
     
     SCHOOL CONTEXT FROM DOCUMENTS:
     {context}
     """
   
   - After receiving Groq response:
     a. Check if response contains "ESCALATE"
     b. Remove the word "ESCALATE" from the response text
     c. If escalation needed:
        - Send CallMeBot WhatsApp alert: GET request to 
          https://api.callmebot.com/whatsapp.php with params:
          phone=CALLMEBOT_PHONE, text="NAWIS AI Alert: A visitor 
          needs help answering: [first 100 chars of user question]. 
          Please come to reception. Time: [current time]", 
          apikey=CALLMEBOT_APIKEY
        - Send ESP32 character "E"
        - Return JSON: { "reply": cleaned_response, 
                         "escalate": true }
     d. If no escalation:
        - Send ESP32 character "S" (speaking)
        - Return JSON: { "reply": cleaned_response, 
                         "escalate": false }

3. GET /status endpoint:
   - Returns JSON: { "status": "ready", "docs_loaded": count }

4. POST /esp32/{state} endpoint:
   - Accepts state: "idle", "listen", "think", "speak", "error"
   - Maps to characters: I, L, T, S, E
   - Sends to ESP32 via serial

5. Serve the frontend:
   - Serve "frontend/index.html" at the root path "/"
   - Serve everything in "frontend/" as static files

════════════════════════════════════
FRONTEND — frontend/index.html
════════════════════════════════════
Design a beautiful, modern, kiosk-style single page. 
VISUAL STYLE:
- Background: clean white (#FFFFFF) and very light gray (#F8F9FA)
- Primary color: deep school green (#1B6B2F)
- Accent: gold (#C49A22)  
- Text: near-black (#1A1A1A) and medium gray (#6B7280)
- Font: Inter (Google Fonts) — clean, modern, professional
- Feel: like a premium Apple/Google product, NOT like a sci-fi AI
- Zero scrolling — everything fits on one screen
- Rounded cards, subtle shadows, clean spacing

LAYOUT (top to bottom, all fitting in 100vh):
1. TOP BAR (compact, ~60px):
   - Left: school logo placeholder (green circle with "N" 
     monogram) + "NAWIS AI" text + small subtitle 
     "Al Wurood International School"
   - Right: connection status dot (green = ready), 
     language toggle button (EN / عربي), 
     voice toggle button (🔊 on/off)

2. CENTRAL ORB SECTION (~200px):
   - A smooth circular orb, 140px diameter
   - Default state: gentle green-to-white gradient, 
     slow breathing pulse animation
   - Listening state (class "listening"): 
     blue-green color, faster pulse, sound wave rings expanding
   - Thinking state (class "thinking"): 
     gold/amber color, rotating ring around orb
   - Speaking state (class "speaking"): 
     green, gentle oscillating scale animation
   - Escalation state (class "escalating"): 
     soft red/orange, slow pulse
   - Below orb: status text label ("Ready to help", 
     "Listening...", "Thinking...", "Speaking...", 
     "Connecting you to staff...")
   - Below status: small mic button (tap to speak) with 
     clean icon — when recording, button turns red with pulse

3. CHAT AREA (flex-grow, scrollable internally):
   - Welcome state (no messages yet): 
     centered greeting text "How can I help you today?" 
     + subtitle in both English and Arabic
     + 8 FAQ chips in a wrapping row (tappable, rounded pill buttons)
       Chips: "Admission process", "School timings", 
       "Available programmes", "Transport & buses",
       "CBSE curriculum", "Contact us", 
       "Exam & results", "School facilities"
   - Chat state (messages exist): 
     standard chat bubbles, user on right (green bubble), 
     AI on left (white card with green left border)
     AI messages have the NAWIS AI logo/icon on the left
     Typing indicator (3 dots animation) while waiting

4. INPUT BAR (compact, ~70px, always visible at bottom):
   - Text input field (rounded, full width with padding)
   - Placeholder: "Type your question or tap the mic..."
   - Send button (green arrow button)
   - Mic button (separate, green mic icon)
   - Input bar has a clean subtle top border separator

VOICE FUNCTIONALITY:
- Use Web Speech API for BOTH STT and TTS
- STT: webkitSpeechRecognition, language switches with EN/AR toggle
  - English: 'en-US', Arabic: 'ar-SA'
  - On mic tap: start recording, set orb to "listening" state,
    call POST /esp32/listen
  - On result: put transcript in input box, auto-send
  - On error: show toast "Could not hear you, please try again"
- TTS: speechSynthesis API
  - After each AI response arrives, speak it aloud if TTS is ON
  - Set orb to "speaking" state, call POST /esp32/speak
  - When done speaking, set orb back to idle, call POST /esp32/idle
  - TTS toggle button shows 🔊 or 🔇

ESCALATION OVERLAY:
- When API returns "escalate": true, show a full-screen overlay 
  (semi-transparent white, blurred background) with:
  - A soft amber/orange notification icon (bell or person)  
  - Heading: "Connecting you with our staff"
  - Subtext: "The headmaster has been notified and will be 
    with you shortly. Thank you for your patience."
  - Arabic translation below
  - "Back to NAWIS AI" button to dismiss
  - This overlay should look calm and professional, not alarming

ARABIC SUPPORT:
- Language toggle switches between EN and AR
- When AR selected:
  - Input placeholder changes to Arabic
  - FAQ chips show Arabic labels
  - Chat direction flips to RTL (dir="rtl")
  - STT language switches to ar-SA
  - TTS language switches to ar-SA
  - All UI labels switch to Arabic

ERROR HANDLING:
- If backend is not reachable: show a toast notification 
  "Connection issue — please try again"
- If mic permission denied: show toast explaining how to allow
- All errors are silent toasts, never crash the whole UI

ADDITIONAL FILES TO CREATE
════════════════════════════════════
1. requirements.txt with all needed packages
2. README.md with:
   - How to add school documents to school_data/ folder
   - How to run the app: "uvicorn app:app --host 0.0.0.0 --port 8000"
   - How to open: go to http://localhost:8000 in Chrome
3. .env.example showing what goes in config.py
4. esp32_code/nawis_led.ino — Arduino code for ESP32 that:
   - Reads single characters from Serial (115200 baud)
   - Controls 5 LEDs on pins 12, 13, 14, 15, 16
   - I = pin 12 GREEN on, rest off (idle)
   - L = pin 13 BLUE/WHITE on fast blink, rest off (listening)
   - T = pin 14 YELLOW/AMBER blink, rest off (thinking)  
   - S = pin 12 GREEN steady + short beep on pin 26 (speaking)
   - E = pin 16 RED blink + urgent beep pattern on pin 26 (escalate)
   - Buzzer on pin 26
   - Default to idle state on startup

IMPORTANT CONSTRAINTS
════════════════════════════════════
- The whole app must start with ONE command: 
  uvicorn app:app --host 0.0.0.0 --port 8000 --reload
- No separate frontend build step — the HTML is static
- Graceful fallback if ESP32 not connected (just skip serial)
- Graceful fallback if school_data folder is empty 
  (AI still works, just without RAG context)
- All API keys in config.py — no user-facing settings panel
- Mobile-friendly layout (in case viewed on phone)
- App should feel FAST — no loading screens, instant UI response
